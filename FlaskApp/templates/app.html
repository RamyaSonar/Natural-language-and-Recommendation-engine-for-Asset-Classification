ASSET CLASSIFICATION AND RECOMMENDATION SYSTEMS USING NATURAL LANGUAGE PROCESSING

Problem Statement

Develop a classification model that recommends the asset classes based on the freeform title, description, and other relevant procurement fields to provide data fidelity. The data comprises of procurement, distribution, maintenance, and retirement of the assets for a period of time. The aim is to be develop a classification model to provide an automatic mapping of order details to the Asset class.

Product Vision - Sample scenarios

Scenario #1

The classification application can be used by a Supply chain organization to precisely assign classes to the orders based on the textual data provided by the users. These organizations prioritize efficient delivery of the products to the customers. It is a dynamic way to classify the orders as the misclassification rate can be decreased. Unlike the traditional way of manually assigning classes to the orders, this approach automates the process, lowering the human effort involved. Our product is more precise and can be used across similar datasets and is not restricted to only one organization. It needs to be noted that model performance is dependent on the quality and the quantity of the data.

Scenario #2

The application or investigation process during the development can be used to identify the classes that are closely related to each other. This involves knowledge obtained using the features, patterns, and model. These insights can be used to combine or remove certain classes or suggest new ones. This will be very useful for various stakeholders in the supply chain cycle to improve or modify the asset classes.

Scenario #3

The product can be used in providing recommendations to the user. Relevant fields are also provided to the user to input content based on the recommended class. This improves the quality of data being generated. Over time, this is expected to increase the fidelity of the data.

Algorithms and Models

Shallow Learning Algorithms

Naïve Bayes

Naïve Bayes uses the probability calculated from the training set to calculate the posterior probability. As shown in Figure 17, y is the target class and [x1,x2 .. xn] are features provided as the input to the model. Naïve Bayes uses the feature selection score method by calculating the KL-Divergence between training documents and their labels. It tries to identify the difference in the probability distribution of one record from another to create a better performing model on the lesser number of classes. It assumes that the target label is assigned based on conditions between the attributes and that all thefeatures are independent. This assumption might not hold in real life.

K-Nearest Neighbors

KNN is an unsupervised classification algorithm, which classifies the new record based on k training sets and a similarity measure (computed based on the Euclidean distance). It is a simple, fast processing model that does not have a training phase. It is called a lazy learner because it does not train a model based on the training data but only memorizes the dataset. However, it was used in pattern recognition and statistical estimation in the early 1970s.

Decision Trees

A decision tree is a supervised learning model which uses a divide and conquers approach to create a tree with the given data. The correlation between the target class and the features is determined by constructing a tree. It uses various tree pruning methods to reduce noise. The complexity of the tree increases with the increase in the size of the dataset. To decrease the computational costs, fast Decision Trees are introduced which uses two-pronged strategies (preselecting the set of features and training a set of DTs on different subsets of data).

Random Forest

Random Forest is a classification algorithm, that creates a set of decision trees. It uses feature randomness and bagging to create a forest of trees. The result is based on a prediction of each tree in the committee. The main limitation of the random forest is that algorithm is too slow because it tries to build a large number of trees and also tries to overfit the data. Random forest aggregates result from a set of decision trees.

Deep Learning Models

Multi-Layer Perceptron(MLP)

MLP is a feed-forward neural network that connects multiple layers in a directed graph. This approach uses a supervised algorithm called the backpropagation technique for training. It has an input hidden layer with an activation function and output layers. The data is fed into the inpullayer and predictions are made at the output layer.

The value at each layer is calculated based on the value in the previous layers and by applying the weights. Every node (the computational unit) in the hidden layer uses an activation function to calculate a singular value at the node. The results at the hidden layer are computed, aggregated and the error rate is calculated at the output layer. The error rate at the previous epoch (train) is used to tune the weights of the current epoch and follow the same process until the termination condition is reached.For text classification, it considers the input text as the BOW.

Convolutional Neural networks

In the beginning, CNNs (Convolutional Neural Networks) were introduced for Image classification, but later because of its convolving filters, this concept has performed better on NLP techniques as well. CNN can apply convolutions on multiple chunks of sequence data simultaneously. Like in image representation, the text is represented in vector format to extract features from different angles. The final vector representation of the text is generated from the convolutional and pooling layer. The word vectors created from the text are joined into a matrix and are fed into the convolutional layer. The convolutional layer contains several filters to process these matrices and the results are sent into the pooling layer and combine the results of the pooling layer to obtain the final vector representation of the text. The target class is predicted using this final vector.

The CNNs have better performance in extracting position-invariant and local features, their processing times are better compared to RNNs. But the model has poor interpretability, and the classification errors cannot be explained because of the non-readability nature of the hidden states. So, Attention nets are introduced to get a better interpretation of the results by employing informational components like encoders, attention layers of a text.

Long Short-term Memory (LSTM)

LSTM contains a gate structure with input, forget, and output gates. These are used to create a cell that remembers the value. The forget gate structure is used to remove the unwanted information and establish a connection among the context words that can be captured by the LSTM classification method. To improve the performance of this sequential classification model, Bi-LSTMs are introduced. This solves the problem by processing the sequence in both directions. Bi-LSTM duplicates the first recurrent layer in the architecture so that there are two layers side-by-side. The input sequence is provided to the first layer and a reversed copy of the sequence to the second layer to extract contextual features from both sides, left-to-right, and right-to-left. As the model is trained numerous times with many features, they are prone to overfitting and it is difficult to apply a dropout algorithm to curb this issue.

Recurrent Neural Networks

Recurrent Neural Networks is a layer made of memory cells that captures the context of the data, required to make predictions by learning the long-term dependencies. At each element of the sequence, the model also considers the information from the preceding elements and along with the current input. RNN considers the location information of the words to learn historical information, syntax and semantics of text automatically without the feature engineering step.The architecture of the Recurrent Neural Network Model, where word embeddings vector is input to the model. These embedding vectors are passed one by one to the model to capture the sequential information. RNN shares parameters at every epoch, to reduce the computational cost of training a model.

To reduce the time complexity of the model, semantics are extracted from long texts. These experiments use relative tasks for exploring common features. The computation of this neural network and training is relatively very slow. The weights in the model are calculated using the derivatives and the gradientsare adjusted using these weights. RNN face the vanishing gradient problem because of the smaller gradient values.

Bidirectional Recurrent Neural Networks (BiRNN)

Bidirectional RNNs are RNNs that offer look-ahead ability as in hidden Markov models by putting two RNNs together. BiRNN runs in two different modes one in the forward mode starting from the first token and the other from the last token running from the back to front. For a more flexible process, the hidden layers in Bidirectional RNNs pass the information in the backward direction. Using Bidirectional RNN, inputs will run in twodifferent ways, one from past to future and the other from future to past.

Gated Recurrent Units (GRU)

Gated Recurrent Units are improved version of Recurrent Neural Networks that solves the vanishing gradient problem. To decide the information that should be passed to output, GRU uses two different vectorsnamely update and reset gates. The update and reset gates are trained to remember the information from previous epochs and remove the irrelevant information to generate predictions. The update gate alsocalle as long-term memory controls the flow of information into memory and reset gate also called as short-term memory controls the flow out of memory.

The update gate is powerful for training a model because it eliminates the risk of gradient vanishingproblem by helping the model understand how much of information from previous epochs is necessary to bepassed to the future. The reset gate eliminates the irrelevant information in the epochs. GRU has only one hidden state unlike LSTM that captures both the long term and short term dependencies. The Bidirectional Gated Recurrent Unit (BiGRU) has two GRUs, one takes inputin backward direction and other in forward direction.
